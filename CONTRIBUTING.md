# Contributing to ComplyFlow

Thank you for your interest in contributing! Please follow these guidelines to help us maintain a safe, open, and high-quality project.

## How to Contribute

- **Fork the repository** and create your branch from `main`.
- **Write clear, well-documented code** and add comments where helpful.
- **Do not include any credentials, secrets, or sensitive data** in your code or configuration files.
- **Update documentation** as needed for your changes.
- **Test your code** before submitting a pull request.
- **Open a pull request** with a clear description of your changes and why they are needed.

## Code of Conduct

- Be respectful and inclusive.
- No harassment, discrimination, or inappropriate behavior.
- See [CODE_OF_CONDUCT.md](CODE_OF_CONDUCT.md) for details.

## Reporting Issues

- Use the [GitHub Issues](../../issues) page to report bugs or request features.
- Provide as much detail as possible to help us reproduce and fix the issue.

## Security

- If you discover a security vulnerability, please report it privately to the maintainers.
- Do not disclose security issues publicly until they are resolved.

---

Thank you for helping make ComplyFlow better!
